# InvenTrack — Inventory & Order Management System

A full-stack system for managing products, customers, and orders, with automatic
inventory tracking. Built with **FastAPI**, **React**, and **PostgreSQL**, fully
containerized with **Docker Compose**.

## Stack

| Layer    | Technology               |
|----------|--------------------------|
| Frontend | React 18 (Vite)          |
| Backend  | FastAPI + SQLAlchemy     |
| Database | PostgreSQL 16            |
| Infra    | Docker, Docker Compose   |

## Project structure

```
inventory-order-system/
├── backend/        # FastAPI service
├── frontend/       # React (Vite) app
├── docker-compose.yml
└── .env.example    
```

## Running locally with Docker Compose

```bash
cp .env.example .env       
docker compose up --build
```

- Frontend: http://localhost:3000
- Backend API: http://localhost:8000
- Interactive API docs (Swagger UI): http://localhost:8000/docs

## Running without Docker (for development)

**Backend**
```bash
cd backend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # point DATABASE_URL at a local Postgres instance
uvicorn app.main:app --reload
```

**Frontend**
```bash
cd frontend
cp .env.example .env
npm install
npm run dev
```

## Business rules implemented

- Product SKUs must be unique
- Customer emails must be unique
- Stock quantity and price can never be negative
- Orders are rejected with `400` if requested quantity exceeds available stock
- Placing an order automatically deducts stock; cancelling an order restores it
- Order total is always calculated server-side, never trusted from the client

## API summary

| Method | Path                | Description                       |
|--------|---------------------|------------------------------------|
| POST   | `/products/`        | Create product                     |
| GET    | `/products/`        | List products                      |
| GET    | `/products/{id}`    | Get product                        |
| PUT    | `/products/{id}`    | Update product                     |
| DELETE | `/products/{id}`    | Delete product                     |
| POST   | `/customers/`        | Create customer                    |
| GET    | `/customers/`        | List customers                     |
| GET    | `/customers/{id}`    | Get customer                       |
| DELETE | `/customers/{id}`    | Delete customer                    |
| POST   | `/orders/`           | Create order (validates + deducts stock) |
| GET    | `/orders/`           | List orders                        |
| GET    | `/orders/{id}`       | Get order with line items          |
| DELETE | `/orders/{id}`       | Cancel order (restores stock)      |
| GET    | `/dashboard/`         | Summary stats for the dashboard    |

Full interactive documentation is auto-generated by FastAPI at `/docs`.

## Deployment

- **Database**: a free hosted Postgres instance (e.g. Render Postgres or Neon)
- **Backend**: deployed as a Docker container on Render
- **Frontend**: deployed as a static build on Vercel, with `VITE_API_URL` set to the live backend URL

After deployment, update the backend's `CORS_ORIGINS` environment variable to the
live frontend URL so the two can talk to each other.
